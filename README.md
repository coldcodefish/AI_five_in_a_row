PS: 带有禁手的五子棋介绍  
三种主要禁手类型：  
- 三三禁手，指黑棋在一子落下后，同时形成两个或两个以上的活三。活三是指三子连线两端为空或间隔一子可形成活四的棋形。若通过连续多步才形成两个活三，则不算禁手 。
- 四四禁手，指黑棋在一子落下后，同时形成两个或两个以上的“四”。“四”包括活四（两端无阻挡）和冲四（一端被挡）。同样，必须是同一次落子形成的双四才构成禁手。
- 长连禁手，指黑棋形成六个或以上连续同色棋子（超过五连）。如果黑棋同时形成五连和禁手形，则禁手失效，以五连胜利为准 。
  
黑棋的合法取胜方式：  
在有禁手规则的比赛中，黑棋不能随意形成禁手棋形取胜。黑棋唯一合法的取胜方式是通过形成“四三”（同时形成一个四和一个活三）来获胜。

## 博弈性说明
### 策梅洛定理(博弈论中一个定理)***[6]***
策梅洛定理针对的是满足以下五个严格条件的博弈：
1. 双人博弈（只有两位玩家）；
2. 零和博弈（一方赢则另一方输，没有平局之外的第三种结果）；
3. 完全信息（双方都能看到棋盘上的所有信息，不存在隐藏牌或暗棋）；
4. 确定性（没有任何运气成分，如掷骰子或洗牌）；
5. 有限步数（游戏必须在有限的步数内结束，不能无限进行下去，且不允许平局，或有明确的平局判定）。

对于满足上述条件的游戏，策梅洛定理断言以下三种可能性必有且仅有一种成立：  
- 先手必胜（先手方存在一套策略，无论后手怎么走，最终都能赢）；
- 后手必胜（后手方存在一套策略，无论先手怎么走，最终都能赢）；
- 双方必和（双方都存在一套策略，保证自己至少不会输，且对方也无法赢）

### 五子棋中策梅洛定理的说明
策梅洛定理在五子棋中所对应的就是：先手(黑棋)必胜。无论有没有禁手的五子棋。  
但依旧不妨碍不知棋谱的人，享受简单思考的快乐，以及AI的相关的无棋谱训练。  

### 分享内容(没有找到出处)
策梅洛定理可以看作是一句充满哲学意味的数学判词：  
在规则确定的有限棋盘上，输赢从第一手落下之前，就已经在数学上被写好了。  
棋手们做的所有努力，本质上都是在这个“已被数学锁定”的结果中，  
尽力寻找那条通向胜利的路线，并诱使对手偏离最优解。

## 算法设计
### 随机算法
随机走子 AI 的原理就是：列出所有空位，闭着眼睛随便挑一个下。没有任何策略，纯粹作为测试其他AI时的“最低水平对手”。

### 强化学习DQN(Deep Q-Network)算法
DQN 用神经网络直接学习每个局面的 Q 值（动作价值），从而实现“端到端”的对弈策略

1. 网络结构
- 输入：棋盘状态（两个通道，分别表示 AI 棋子和对手棋子）。
- 输出：每个空位的 Q 值，代表在该位置落子的期望未来总回报。
使用两个结构相同的网络：评估网络（决定走棋）和目标网络（计算目标 Q 值），定期同步参数以保证训练稳定。

2. 训练过程
AI 自我对弈，每一步将 (状态, 动作, 奖励, 下一状态) 存入经验回放池。
随机抽取一批经验，计算目标 Q 值：
Q_target = 即时奖励 + γ × max(目标网络对下一状态的 Q 值)
让评估网络的输出拟合该目标，反向传播更新参数。
初期以 ε-贪心策略随机探索，后期逐步降低探索率，根据最高 Q 值选步。
训练完成后，推理时直接选择 Q 值最大的合法空位落子，不再探索。 

### 蒙特卡洛树搜索（MCTS）
蒙特卡洛树搜索（MCTS）的核心是：不依赖人工评估函数，通过大量随机模拟“试出”每步棋的好坏。每次模拟包含四步：
- 选择：从根节点开始，用 UCB 公式（平衡胜率和探索）选一个最有潜力的节点。
- 扩展：在该节点上随机选一个没试过的空位，生成新子节点。
- 模拟：从新节点开始，双方随机下到终局，记录谁赢。
- 回溯：把胜负结果沿着路径传回去，更新沿途节点的访问次数和胜场。

重复几百次后，AI 选择根节点下访问次数最多的子节点作为最终落子（访问次数多意味着结果更可信）。它不搜索全部未来，而是用随机采样代替精确计算，适合分支庞大的棋类。

### 贪心评分算法
把每个空位当成落子点，看它周围能形成多长的连续棋子和几个开放口，根据固定表格打分，最后选分最高的位置下。  
评分方法：
贪心评分算法逐个遍历空位 (r, c)，分别计算进攻分和防守分：
- 进攻分：假设 AI 在此处落子，统计四个方向上能形成的最强棋型，累加分数。
- 防守分：假设对手在此处落子，同样统计其威胁程度。
最后总分 = 进攻分 × 1.1 + 防守分（略微偏重进攻）。选择总分最高的点作为最佳落子。

### 博弈树极大极小值alpha-beta剪枝搜索
- 博弈树：把未来所有可能的下棋顺序画成一棵树。
- 极大极小：AI假设对手下得最好，然后在这种最坏情况下，为自己挑一个最好的结果。
- Alpha‑Beta剪枝：搜索时一旦发现某个走法肯定不如已经找到的，就立刻跳过，省时间。
- Negamax：因为“对你好就是对我坏”，所以把极大和极小合并成一个函数，每层都取负号就行。
- 评估函数：给棋盘打分，看谁的连线更厉害（比如活四很高分，活二很低分）。

具体说明：
1. 博弈树
将当前局面作为根节点，所有合法走法作为第一层分支，对手的回应作为第二层分支，AI 的再回应作为第三层……如此构成一棵树。

2. 极大极小值原理
AI 假设对手同样会走出最优棋步：
- 极大层（AI 回合）：在所有候选走法中，选择让评估分数最大的那个。
- 极小层（对手回合）：在所有候选走法中，选择让评估分数最小的那个（因为分数是对 AI 有利程度的度量）。
这样一层层倒推，AI 便能在最坏情况下找出对自己最有利的走法。

3. Alpha-Beta 剪枝
搜索过程中维护两个边界：
- α：当前路径上极大层已知的最大值下界。
- β：当前路径上极小层已知的最小值上界。
一旦发现某个分支的分数不可能改善父节点的选择（即 α ≥ β），立即放弃对该分支剩余兄弟节点的搜索，从而大幅减少计算量。

4. Negamax 实现
由于五子棋是零和博弈，一方分数恰好是另一方的相反数。因此可以将极大极小合并为一个函数，每递归一层对分数取负值：
```python
value = -negamax(..., -beta, -alpha)
```
极大层选最大值、极小层选最小值的逻辑被自动统一。

5. 评估函数
当搜索到达预设深度或游戏结束时，调用评估函数为局面打分。评估函数就是上述棋型得分的总和：
```text
总得分 = 己方棋型总分 - 对手棋型总分 × 防守系数
```
这里的防守系数可调节攻守倾向。
